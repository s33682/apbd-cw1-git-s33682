# 1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?

Fast-Forward kiedy main nie został ruszony przy zmianach na pobocznej gałęzi
Merge commit kiedy main został zmieniony przy zmianach na pobocznej gałęzi

# 2. Czym w praktyce różni się merge od rebase? 

Merge zostawia po sobie ślad (commit) natomiast rebase jest dołączany po cichu

# 3. W jaki sposób został rozwiązany konflikt w Twoim repozytorium?
